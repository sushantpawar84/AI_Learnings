DATABASE SCHEMA
================================================================================

Three tables track migration state:

Table: migration_metadata
  - id (PRIMARY KEY)
  - svn_url
  - git_url
  - start_time
  - last_update_time
  - last_checkpoint_revision (used for resume)
  - total_svn_revisions
  - current_batch_number
  - migration_status

Table: migration_checkpoints
  - id (PRIMARY KEY)
  - svn_revision (UNIQUE) â† Resume from MAX(this) WHERE status='completed'
  - git_commit_sha
  - svn_author
  - svn_timestamp
  - svn_message
  - batch_number
  - created_at
  - updated_at
  - status (pending, in_progress, completed, failed, skipped)
  - error_message
  - retry_count

WORKFLOW DIAGRAM
================================================================================

First Run:
  1. Initialize database
  2. Query SVN: total revisions = 1000
  3. Fetch revisions 1-50
  4. Create Git commits 1-50
  5. Push batch to GitHub
  6. Mark 1-50 as completed in DB
  7. Continue to revisions 51-100
  8. ... repeat until done

Resume After Failure:
  1. Query DB: last completed = 750
  2. Fetch revisions 751-800
  3. Create Git commits 751-800
  4. Push batch to GitHub
  5. Mark 751-800 as completed in DB
  6. Continue to revisions 801-850

Result: Zero duplicate commits, resume from exact point!

================================================================================
PERFORMANCE CHARACTERISTICS
================================================================================

Default Settings:
  - Batch Size: 50 revisions
  - Timeout: 300 seconds per batch
  - Retry Attempts: 3 with exponential backoff (5s, 15s, 45s)
  - Database: SQLite (<1ms per query)
  - Memory: Low overhead, streaming operations

Typical Throughput:
  - Small repos (< 1k commits): 5-10 minutes
  - Medium repos (10k commits): 1-2 hours
  - Large repos (100k+ commits): 8-24 hours
  - Very large repos: May need configuration tuning

Bottlenecks (in order of impact):
  1. SVN fetch speed (depends on server)
  2. Git push speed (depends on network)
  3. Git commit creation (sequential)
  4. Database writes (SQLite, fast)
